EverestOS Developer Guides

Welcome to the official development documentation for EverestOS — a premium, modular web-based desktop environment born in Nepal. This guide covers building applications, extensions, and working with the system APIs.

🌿 Inspired by Cinnamon Desktop

EverestOS draws heavy inspiration from the Cinnamon Desktop Environment by the Linux Mint team. We've ported the core extension concepts — Applets, Desklets, and the Settings Schema system — to the web using a custom CommonJS loader and mock St (Shell Toolkit) libraries.

1. Building Applications

Applications are the primary extension mechanism in EverestOS. Each app is a self-contained module with a manifest and entry point.

App Directory Structure

~/Apps/my-app/
├── app.json # Manifest (required)
└── app.js # Entry point (required)

Manifest (app.json)

{
  "id": "my-app",
  "name": "My Application",
  "description": "A sample EverestOS application",
  "icon": "archive",
  "category": "Utilities",
  "version": "1.0.0"
}
FieldRequiredDescription
id✅Unique identifier (kebab-case)
name✅Display name shown in App Menu
description❌Short description
icon❌Bloom theme icon key or emoji (e.g., "folder", "📦")
category❌Menu category: Utilities, Development, System, Multimedia, Internet, Office
version❌Semantic version string

Entry Point (app.js)

Your app must export an async launch function that receives the system context:

export async function launch(ctx, options = {}) {
  const { windowManager, vfs, appLoader } = ctx;

  // Create your UI
  const content = document.createElement('div');
  content.style.cssText = 'padding:20px; color:var(--text-primary);';
  content.innerHTML = '<h1>Hello EverestOS!</h1>';

  // Open a window
  windowManager.createWindow({
    id: 'my-app-' + Date.now(),
    title: 'My Application',
    icon: 'archive',
    width: 600,
    height: 400,
    content
  });
}

Context Object Properties

PropertyTypeDescription
windowManagerWindowManagerCreate, close, minimize, and focus windows
vfsVirtualFileSystemRead/write the virtual filesystem
appLoaderAppLoaderLaunch other applications programmatically
panelManagerPanelManagerAccess the taskbar and system tray
filePickerFilePickerAppOpen file/folder selection dialogs
consoleLookingGlassLog messages to the Looking Glass console
themeManagerThemeManagerGet current theme information
showSystemDialogFunctionDisplay alert/confirm dialogs

💡 Launching Other Apps

You can launch any registered app from your code: ctx.appLoader.launchApp('web-browser', { url: 'https://example.com' })

2. Building Applets Panel

Applets are panel widgets that use the Cinnamon-compatible CommonJS pattern with require().

Applet Structure

~/Plugins/applets/my-applet@author/
├── metadata.json # Extension metadata
├── applet.js # Entry point
└── settings-schema.json # Optional settings

Example Applet

const Applet = require('ui/applet');
const St = require('gi/st');
const Main = require('ui/main');

class MyApplet extends Applet.TextIconApplet {
  _init(orientation, panelHeight, instanceId) {
    super._init(orientation, panelHeight, instanceId);
    this.set_applet_icon_name('preferences-system');
    this.set_applet_label('Hello');
    this.set_applet_tooltip('My custom applet');
  }

  on_applet_clicked() {
    Main.notify('Applet clicked!');
  }
}

function main(metadata, orientation, panelHeight, instanceId) {
  return new MyApplet(orientation, panelHeight, instanceId);
}

3. Building Desklets Desktop

Desklets are floating desktop widgets. They can be dragged, configured, and persist across sessions.

const Desklet = require('ui/desklet');
const St = require('gi/st');

class MyDesklet extends Desklet.Desklet {
  _init(metadata, instanceId) {
    super._init(metadata, instanceId);
    this.setupUI();
  }

  setupUI() {
    const label = new St.Label({
      text: 'Hello Desktop!',
      style: 'font-size: 24px; color: white;'
    });
    this.setContent(label);
  }

  on_desklet_removed() {
    // Cleanup timers, intervals, etc.
  }
}

function main(metadata, instanceId) {
  return new MyDesklet(metadata, instanceId);
}

4. Virtual File System API

The VFS provides a unified filesystem interface that works in both development (Node.js backend) and production (IndexedDB) modes.

const vfs = ctx.vfs;

// Read directory contents
const items = await vfs.readdir('/home/user/Documents');

// Read a file
const text = await vfs.readFile('~/Documents/notes.txt');

// Write a file
await vfs.writeFile('~/Documents/notes.txt', 'Hello!');

// Check if path exists
const stat = await vfs.stat('~/Documents');

// Create a directory
await vfs.mkdir('~/Documents/new-folder');

// Copy, rename, delete
await vfs.copy('~/file.txt', '~/Desktop/file.txt');
await vfs.rename('~/old.txt', '~/new.txt');
await vfs.rm('~/temp.txt');
await vfs.trash('~/unwanted.txt');

📁 Path Aliases

~ resolves to /home/user. All paths are absolute within the VFS namespace.

6. Icon Helper API

Always use IconHelper for theme-aware icon rendering:

// Theme icon with emoji fallback
IconHelper.getIcon('folder,📁', { size: 24 });

// Fixed path (ignores theme)
IconHelper.getIcon('/icons/everest-logo.svg', { size: 64 });

Icon keys use comma-separated format: 'bloom-theme-key,emoji-fallback'

7. Theme System

Themes are JSON files in public/themes/ that define CSS variables and icon packs:

{
  "name": "Mint Dark",
  "type": "dark",
  "icon-theme": "bloom-dark",
  "colors": {
    "--bg-primary": "#1a1a2e",
    "--text-primary": "#e8e8e8",
    "--mint-green": "#6ecf8a"
  }
}

Key CSS variables: --bg-primary, --bg-surface, --bg-card, --text-primary, --text-secondary, --text-accent, --border, --mint-green.
