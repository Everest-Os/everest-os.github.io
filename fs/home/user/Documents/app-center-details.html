✨ Platform Features

⚡ Native Performance

Built with vanilla JavaScript and modern web standards — no heavy frameworks, maximum speed.

🔒 Sandboxed Execution

Every application runs in isolation. Extensions use a controlled CommonJS runtime environment.

🎨 Adaptive Theming

Full light/dark mode with 600+ high-fidelity Bloom SVG icons that adapt to your theme automatically.

💾 Virtual File System

A complete filesystem abstraction supporting IndexedDB and native Node.js backends.

🧩 Extension System

Cinnamon-compatible applets and desklets powered by a custom CJS loader and mock St libraries.

🛠️ Developer Tools

Built-in Developer Center, Looking Glass console (Alt+F2), and live code editor for rapid prototyping.

🧩 System Extensions

Beyond standalone apps, EverestOS supports three types of extensions that integrate directly into the desktop shell — modeled after the Cinnamon Desktop Environment.

📌 Applets — Panel Widgets

Small tools that live in the taskbar panel. Examples: clocks, date displays, system monitors, and the main application menu.

Location: ~/Plugins/applets/<uuid>/

🖥️ Desklets — Desktop Widgets

Floating, draggable widgets on the desktop surface. Examples: seven-segment clocks, calendar displays, system info panels.

Location: ~/Plugins/desklets/<uuid>/

📱 User Applications

Custom applications installed by the user or through the App Center. These appear in the App Menu and can be uninstalled.

Location: ~/Apps/<app-name>/

📥 How to Install & Manage Apps

Installing

  • Open the App Center from the App Menu
  • Browse the Curated or Installed tabs
  • Click Install on any application card
  • The app will appear in your App Menu immediately

Uninstalling

  • Open the App Center and go to the Installed tab
  • Click Uninstall on any user-installed application
  • Built-in system applications cannot be uninstalled

Managing Extensions

  • Open Plugin Settings from the App Menu or System Settings
  • Toggle extensions on or off with the enable/disable controls
  • Right-click any applet or desklet for context menu options (Settings, About, Remove, Reload)

💡 Creating Your Own Apps

Use the Developer Center to create, edit, and export custom applications. Each app needs just two files: app.json (manifest) and app.js (entry point). See the Developer Guide for full API documentation.

🚧 Coming Soon

The App Center install functionality for remote/community apps currently operates as a UI preview. A centralized repository server for one-click community app installations is planned for a future release.
